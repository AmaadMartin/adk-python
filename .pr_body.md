**Problem**: In workflow node runners, `event.author` was blindly overwritten by the outer workflow's context event author, erasing the actual entity that generated the content.

**Solution**: Updated `_node_runner.py` to preserve an explicitly set `event.author`. Also added inference to set `event.author = 'user'` when the event has no author but includes content with `role == 'user'`, defaulting to the workflow context author only when necessary. This results in accurate telemetry, tracing, and replay behaviors.
